# PRD: Daily Inbox 个人日常记录与待办整理工具

## 1. 背景

用户日常会产生大量零散信息，包括学习记录、工作事项、生活琐事、突然想到的灵感、临时待办、未来计划等。

这些信息往往出现得很随机：有时只是想记一句话，有时是之后要做的事，有时带有 deadline，有时只是一个模糊想法。传统待办工具要求用户先分类、填写字段、设置时间，记录成本较高；日记工具又缺少待办跟进和自动整理能力。

因此需要一个低摩擦的个人记录工具：用户只负责随手输入，系统自动完成分类、标签、待办识别、时间提取，并生成每日/每周总结。

## 2. 产品定位

Daily Inbox 是一个面向个人使用的 H5 + 后台管理工具。

用户可以在 H5 端快速记录任意内容，系统自动判断内容属于学习、工作、生活、想法或待办，并识别是否需要跟进、是否存在 deadline 或计划时间。后台用于管理记录、待办、分类、标签和总结内容。

## 3. 目标

本项目第一版主要用于验证一套 AI 参与开发的完整流程，同时产出一个用户本人可以长期使用的 MVP。

核心目标：

1. 降低日常记录成本。
2. 自动整理零散记录。
3. 提取需要后续处理的事项。
4. 支持每日和每周总结。
5. 提供后台管理能力，方便修正和查看数据。

## 4. 用户角色

第一版仅面向单用户使用。

角色：

- 普通用户：在 H5 端记录内容、查看今日记录、查看待跟进事项和总结。
- 管理员：在后台管理全部记录、分类、标签、待办和总结。

第一版普通用户和管理员可以是同一个人。

## 5. 使用场景

### 场景 1：随手记录学习内容

用户输入：

```text
今天看了一点 React Server Components，缓存机制还是没完全懂
```

系统自动识别：

- 分类：学习
- 标签：React、Server Components、缓存
- 是否待办：否
- 情绪/状态：困惑

### 场景 2：记录临时待办

用户输入：

```text
明天下午记得改一下登录页按钮样式
```

系统自动识别：

- 分类：工作
- 是否待办：是
- 计划时间：明天下午
- 标签：登录页、按钮样式

### 场景 3：记录有 deadline 的事项

用户输入：

```text
6月30号前把项目的测试验收标准写完
```

系统自动识别：

- 分类：工作
- 是否待办：是
- deadline：2026-06-30
- 标签：测试验收、项目

### 场景 4：记录突然的想法

用户输入：

```text
突然想到可以做一个把 Notion 内容转成小红书卡片的工具
```

系统自动识别：

- 分类：想法
- 标签：Notion、小红书、工具
- 是否待办：否
- 是否值得跟进：是

## 6. 功能范围

第一版包含 H5 端和后台管理端。

H5 端功能：

1. 快速记录
2. 今日记录流
3. 待跟进中心
4. 总结查看
5. 记录详情

后台管理功能：

1. 记录管理
2. 待办管理
3. 分类与标签管理
4. 总结管理
5. AI 识别配置或模拟规则配置

## 7. H5 端需求

### 7.1 快速记录

用户进入首页后，可以直接看到输入框。

功能要求：

- 支持输入文字内容。
- 支持提交记录。
- 提交后系统自动识别分类、标签、是否待办、时间信息。
- 提交成功后，记录出现在今日记录流。
- 输入框提交后清空。
- 内容不能为空。

第一版只支持文字，不支持图片、语音、附件。

### 7.2 今日记录流

展示当天提交的所有记录。

每条记录展示：

- 原始内容
- 分类
- 标签
- 是否待办
- 计划时间或 deadline
- 创建时间
- 当前状态

记录状态包括：

- 普通记录
- 待跟进
- 已完成
- 已归档

### 7.3 待跟进中心

集中展示所有需要后续处理的事项。

筛选维度：

- 今天
- 明天
- 本周
- 逾期
- 无时间
- 全部

待办状态：

- 待处理
- 进行中
- 已完成
- 已取消

用户可以在 H5 端完成以下操作：

- 标记完成
- 修改计划时间
- 修改 deadline
- 将普通记录转为待办
- 将待办取消

### 7.4 总结查看

用户可以查看：

- 今日总结
- 本周总结

今日总结内容包括：

- 今日记录数量
- 各分类数量
- 今日关键词
- 今日主要内容概括
- 待跟进事项
- 逾期事项
- 值得继续推进的想法

每周总结内容包括：

- 本周记录趋势
- 高频分类
- 高频标签
- 主要工作/学习/生活内容
- 本周新增想法
- 本周完成事项
- 下周建议跟进事项

### 7.5 记录详情

用户点击某条记录后，可以查看详情。

详情字段：

- 原始内容
- 分类
- 标签
- 情绪/状态
- 是否待办
- 计划时间
- deadline
- 当前状态
- AI 识别结果
- 创建时间
- 更新时间

支持用户手动修改：

- 分类
- 标签
- 是否待办
- 计划时间
- deadline
- 状态

## 8. 后台管理需求

### 8.1 记录管理

管理员可以查看全部记录。

支持：

- 按日期筛选
- 按分类筛选
- 按标签筛选
- 按是否待办筛选
- 按状态筛选
- 搜索原始内容

管理员可以：

- 查看详情
- 编辑分类
- 编辑标签
- 修改状态
- 删除记录

### 8.2 待办管理

后台展示所有待办事项。

字段包括：

- 内容
- 分类
- 标签
- 计划时间
- deadline
- 状态
- 是否逾期
- 创建时间

支持：

- 修改状态
- 修改时间
- 标记完成
- 取消待办

### 8.3 分类与标签管理

第一版默认分类：

- 学习
- 工作
- 生活
- 想法
- 待办
- 其他

后台支持：

- 查看分类列表
- 新增标签
- 合并标签
- 删除无用标签

分类第一版可以固定，不一定支持新增分类。

### 8.4 总结管理

后台可以查看系统生成的每日总结和每周总结。

支持：

- 按日期查看每日总结
- 按周查看每周总结
- 手动重新生成总结
- 手动编辑总结内容

### 8.5 AI 识别配置

第一版可以采用两种实现方式：

- 真实 AI 接口：调用模型完成分类、标签、待办识别、时间提取和总结。
- 模拟规则：用关键词和规则先跑通流程。

为了 1 天内完成 MVP，建议开发时先保留统一接口，底层可以先使用模拟规则，后续替换为真实 AI。

## 9. 自动识别规则

系统提交记录后，需要输出结构化结果。

字段：

```json
{
  "category": "工作",
  "tags": ["登录页", "按钮样式"],
  "is_todo": true,
  "planned_time": "2026-06-17 14:00",
  "deadline": null,
  "emotion": "平静",
  "follow_up_required": true,
  "summary": "修改登录页按钮样式"
}
```

分类判断：

- 包含学习、看书、课程、教程、研究、理解等，倾向分类为学习。
- 包含项目、需求、代码、页面、测试、客户、会议等，倾向分类为工作。
- 包含吃饭、运动、家务、朋友、照片、生活安排等，倾向分类为生活。
- 包含想到、可以做、灵感、想法、以后做等，倾向分类为想法。
- 包含记得、要做、别忘了、deadline、之前、明天、周末等，倾向识别为待办。

时间识别：

- 支持识别今天、明天、后天、本周、周末、下周。
- 支持识别具体日期，如 6月30号、2026-06-30。
- 支持识别模糊计划，如有空、之后、周末，标记为无精确时间。

## 10. 验收标准

### H5 端验收

1. 用户可以成功提交一条文字记录。
2. 提交后记录出现在今日记录流。
3. 系统可以自动生成分类和标签。
4. 包含“明天”“周末”“6月30号前”等内容时，可以识别为待办。
5. 用户可以在待跟进中心看到待办。
6. 用户可以标记待办为已完成。
7. 用户可以查看今日总结。
8. 用户可以查看本周总结。

### 后台验收

1. 管理员可以查看全部记录。
2. 管理员可以按分类、日期、状态筛选记录。
3. 管理员可以编辑记录分类、标签和状态。
4. 管理员可以查看待办列表。
5. 管理员可以查看每日/每周总结。
6. 管理员可以触发重新生成总结。

### AI/规则验收

1. 输入学习类内容时，分类结果大概率为学习。
2. 输入工作类内容时，分类结果大概率为工作。
3. 输入生活类内容时，分类结果大概率为生活。
4. 输入想法类内容时，分类结果大概率为想法。
5. 输入含时间和动作的内容时，可以识别为待办。
6. 总结内容能覆盖当天主要记录和待跟进事项。

## 11. 非目标

第一版不做：

- 多用户系统
- 社交分享
- 图片记录
- 语音输入
- 日历同步
- 微信提醒
- 复杂权限系统
- 移动端 App
- 高级数据统计
- 完整自然语言时间解析

## 13. 一天开发建议

优先级 P0：

- 文字记录提交
- 自动分类/标签/待办识别
- 今日记录列表
- 待跟进列表
- 标记完成
- 今日总结
- 后台记录管理

优先级 P1：

- 周总结
- 后台总结管理
- 标签管理
- 手动编辑分类和标签

优先级 P2：

- AI Prompt 配置
- 重新生成总结
- 情绪识别
- 更复杂的时间解析

## 14. 成功标准

这个测试项目算成功，如果满足：

1. 你可以在 H5 端顺手记录每天的学习、工作、生活、想法和待办。
2. 系统能自动把大部分记录分到合理分类。
3. 临时事项能进入待跟进中心，不容易丢。
4. 每日总结能帮你回顾今天发生了什么。
5. 这个项目的开发过程能完整验证：PRD → 技术方案 → 开发 → 测试 → 验收。
