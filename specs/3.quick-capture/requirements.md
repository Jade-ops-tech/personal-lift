# quick-capture — 需求规格

## 概述

H5 端快速记录：用户在首页输入框随手输入文字并提交，系统自动识别后写入，记录即时出现在今日记录流。

## 项目信息

- 项目名: personal-lift
- 架构类型: Turborepo monorepo

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-06-17 | v1 | 初始需求 |

## 用户故事

- 作为用户，我想在首页直接看到输入框并随手记一句话，以便用最低成本完成记录。
- 作为用户，我想提交后立刻在今日记录流看到这条记录及其分类/标签，以便确认已记下。

## 功能需求

1. [F-001] 首页输入框：支持输入文字、提交；内容非空校验；提交后清空。
2. [F-002] 提交时调用 2.ai-recognition 识别，结果与原文一并写入 records。
3. [F-003] 今日记录流：展示当天所有记录，每条含原始内容、分类、标签、是否待办、计划时间/deadline、创建时间、当前状态。
4. [F-004] 提交成功后新记录即时出现在今日记录流顶部。

> 第一版仅文字，不支持图片/语音/附件（PRD 7.1）。

## 非功能需求

- 性能: 提交体验流畅；识别较慢时不阻塞 UI 反馈（可乐观插入或加载态）。
- 安全: 写入经 protectedProcedure / 校验输入（security.md）。

## 验收标准

- [ ] [AC-001] 用户可成功提交一条文字记录（PRD H5 验收 1）。
- [ ] [AC-002] 提交后记录出现在今日记录流（H5 验收 2）。
- [ ] [AC-003] 系统自动生成分类和标签并展示（H5 验收 3）。
- [ ] [AC-004] 空内容无法提交。

## 依赖
