# daily-summary — 需求规格

## 概述

生成并展示今日总结：统计当天记录并用 AI 概括主要内容、关键词、待跟进与逾期事项、值得推进的想法。

## 项目信息

- 项目名: personal-lift
- 架构类型: Turborepo monorepo

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-06-17 | v1 | 初始需求 |

## 用户故事

- 作为用户，我想查看今日总结，以便快速回顾今天记录了什么、还有哪些要跟进。

## 功能需求

1. [F-001] 生成今日总结，内容含：今日记录数量、各分类数量、今日关键词、今日主要内容概括、待跟进事项、逾期事项、值得继续推进的想法。
2. [F-002] H5 总结页展示今日总结。
3. [F-003] 总结持久化到 summaries（type=daily, period_key=日期），重复生成覆盖更新。

## 非功能需求

- 性能: 概括类调用允许稍高延迟，提供生成中状态。
- 安全: 生成/查询经 protectedProcedure（security.md）。

## 验收标准

- [ ] [AC-001] 用户可查看今日总结（PRD H5 验收 7）。
- [ ] [AC-002] 总结内容覆盖当天主要记录和待跟进事项（AI 验收 6）。
- [ ] [AC-003] 当天无记录时给出合理空态，不报错。

## 依赖

- 1.data-foundation（records / summaries）
- 2.ai-recognition（复用 Claude 接入做概括，模型可用 claude-opus-4-8）
- 3.quick-capture（记录数据来源）
