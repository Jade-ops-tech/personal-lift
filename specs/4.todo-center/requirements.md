# todo-center — 需求规格

## 概述

H5 待跟进中心：集中展示需后续处理的事项，支持多维筛选、状态流转，以及标记完成、改时间、转待办、取消等操作。

## 项目信息

- 项目名: personal-lift
- 架构类型: Turborepo monorepo

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-06-17 | v1 | 初始需求 |

## 用户故事

- 作为用户，我想在一个地方看到所有待办并按时间筛选，以便不漏掉要做的事。
- 作为用户，我想一键标记完成或调整时间，以便快速维护待办状态。

## 功能需求

1. [F-001] 待跟进列表：展示所有 is_todo 记录。
2. [F-002] 筛选维度：今天/明天/本周/逾期/无时间/全部。
3. [F-003] 待办状态：待处理/进行中/已完成/已取消。
4. [F-004] 操作：标记完成、修改计划时间、修改 deadline、将普通记录转为待办、将待办取消。

## 非功能需求

- 安全: 所有操作经 protectedProcedure 校验归属（security.md）。

## 验收标准

- [ ] [AC-001] 含「明天」「周末」「6月30号前」等内容的记录可在待跟进中心看到（PRD H5 验收 4、5）。
- [ ] [AC-002] 用户可标记待办为已完成（H5 验收 6）。
- [ ] [AC-003] 各筛选维度返回正确集合（逾期 = deadline 过期且未完成）。
- [ ] [AC-004] 普通记录可转为待办、待办可取消，状态正确变更。
