# admin-management-extras — 需求规格

## 概述

后台扩展管理：待办管理、分类与标签管理、总结管理（查看/重新生成/手动编辑），补齐 PRD 8.2/8.3/8.4。

## 项目信息

- 项目名: personal-lift
- 架构类型: Turborepo monorepo

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-06-17 | v1 | 初始需求 |

## 用户故事

- 作为管理员，我想在后台集中管理待办、标签与总结，以便维护数据质量并复核 AI 生成的总结。

## 功能需求

1. [F-001] 待办管理：展示所有待办（内容/分类/标签/计划时间/deadline/状态/是否逾期/创建时间），支持修改状态、修改时间、标记完成、取消。
2. [F-002] 分类与标签管理：查看固定分类列表；标签新增、合并、删除无用标签。
3. [F-003] 总结管理：按日期查看每日总结、按周查看每周总结、手动重新生成、手动编辑总结内容。

## 非功能需求

- 安全: 全部经 protectedProcedure（admin），未登录拒绝（security.md）。

## 验收标准

- [ ] [AC-001] 管理员可查看待办列表并修改状态/时间/完成/取消（PRD 后台验收 4）。
- [ ] [AC-002] 管理员可查看每日/每周总结（后台验收 5）。
- [ ] [AC-003] 管理员可触发重新生成总结（后台验收 6）。
- [ ] [AC-004] 标签可新增、合并、删除，合并后关联记录指向保留标签。

## 依赖

- 5.daily-summary、8.weekly-summary（总结生成能力，供「重新生成」复用）
