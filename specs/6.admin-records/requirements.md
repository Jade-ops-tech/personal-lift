# admin-records — 需求规格

## 概述

后台管理端登录与记录管理：管理员登录后可多维筛选、搜索、查看、编辑、删除全部记录。

## 项目信息

- 项目名: personal-lift
- 架构类型: Turborepo monorepo

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-06-17 | v1 | 初始需求 |

## 用户故事

- 作为管理员，我想登录后台并集中管理全部记录，以便修正分类、标签与状态、清理无用数据。

## 功能需求

1. [F-001] 后台登录页：邮箱+密码登录（better-auth，单用户）。
2. [F-002] 记录管理列表：按日期/分类/标签/是否待办/状态筛选，按原始内容搜索。
3. [F-003] 记录操作：查看详情、编辑分类、编辑标签、修改状态、删除记录。

## 非功能需求

- 安全: 后台所有数据接口为 protectedProcedure，未登录拒绝访问（security.md）。
- 访问控制: 单用户即管理员，登录态校验在服务端。

## 验收标准

- [ ] [AC-001] 管理员可登录后台并查看全部记录（PRD 后台验收 1）。
- [ ] [AC-002] 可按分类、日期、状态筛选记录（后台验收 2）。
- [ ] [AC-003] 可编辑记录分类、标签和状态（后台验收 3）。
- [ ] [AC-004] 未登录访问后台被拦截。

## 依赖

- 1.data-foundation（records、tRPC 基础、better-auth 会话）
- 3.quick-capture（记录数据来源）

## 开放问题

- 无（登录方案已确认 better-auth 邮箱密码）。
