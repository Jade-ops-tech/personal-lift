# record-detail — 需求规格

## 概述

H5 记录详情页：查看单条记录全部字段与 AI 识别结果，并支持手动修改分类、标签、是否待办、时间与状态。

## 项目信息

- 项目名: personal-lift
- 架构类型: Turborepo monorepo

## 需求版本

| 日期 | 版本 | 说明 |
| ---- | ---- | ---- |
| 2026-06-17 | v1 | 初始需求 |

## 用户故事

- 作为用户，我想点开一条记录看到完整信息并在识别有误时手动修正，以便数据更准确。

## 功能需求

1. [F-001] 详情展示：原始内容、分类、标签、情绪/状态、是否待办、计划时间、deadline、当前状态、AI 识别结果、创建时间、更新时间。
2. [F-002] 手动修改：分类、标签、是否待办、计划时间、deadline、状态。
3. [F-003] 从今日记录流/待跟进列表点击进入详情。

## 非功能需求

- 安全: 查询/更新经 protectedProcedure，按归属（security.md）。

## 验收标准

- [ ] [AC-001] 用户可查看某条记录的完整详情与 AI 识别结果。
- [ ] [AC-002] 用户手动修改分类/标签/待办/时间/状态后持久化生效。
- [ ] [AC-003] 修改后返回列表，对应记录展示同步更新。
